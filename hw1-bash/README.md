# 🐚 HW1 — Bash-скрипт настройки пользователей

## Задание

Написать bash-скрипт, который:

1. Создаёт группу `dev`, в которую добавляются все **не-системные** пользователи (UID ≥ 1000)
2. Выдаёт группе `dev` права `sudo` без запроса пароля (через `/etc/sudoers.d/`)
3. Создаёт директории по маске `<user_name>_workdir` для каждого пользователя
4. Путь задаётся ключом `-d`; если не задан — запрашивается интерактивно
5. Права на директории — `660`, владелец — пользователь, группа — группа пользователя
6. Для всех созданных директорий прописывается **ACL на чтение** для группы `dev` (`setfacl -m g:dev:r`)
7. Лог пишется одновременно в `stdout` и в файл `/var/log/setup_users.log`

## Запуск демонстрации

```bash
cd hw1-bash
./run_demo.sh
```

Демонстрация:
1. Собирает Docker-образ с тестовыми пользователями (`alice`, `bob`, `charlie`)
2. Запускает скрипт внутри контейнера с ключом `-d /workspaces`
3. Проверяет результаты: группа `dev`, `sudoers`, директории, ACL

---

## Ручной запуск в Docker

```bash
# Сборка
docker build -t devops-hw1-bash .

# Интерактивная оболочка
docker run --rm -it devops-hw1-bash

# Внутри контейнера:
sudo /usr/local/bin/setup_users.sh -d /workspaces
# или:
sudo /usr/local/bin/setup_users.sh -d /workspaces -n  # dry-run
```

## ⚠️ Замечание о правах 660

В задании указаны права `660` (`rw-rw----`). Для **директорий** такой набор прав означает:

- ✅ Можно читать список файлов (`ls`)
- ✅ Владелец и группа могут читать/писать
- ❌ **Нельзя войти в директорию** (`cd`) — для этого нужен бит execute (`x`)

Скрипт честно выставляет `660`, как того требует задание.

Если требуется, чтобы пользователь мог **работать** внутри директории, права должны быть `770` (`rwxrwx---`).
